# 候选人Excel文件格式说明

## 文件位置

```
assets/lottery/prizeDate.xlsx
```

## Excel 格式要求

### 工作表

- 使用第一个工作表（Sheet1）
- 第一行为标题行，会被跳过
- 从第二行开始为候选人数据

### 列定义

| 列序 | 列名 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| A | 候选人ID | 否 | 唯一标识符 | U100156001 |
| B | 姓名 | **是** | 候选人姓名 | 张三 |
| C | 部门 | **是** | 所属部门 | 开元商铺 |
| D | 备注 | 否 | 附加信息 | VIP客户 |

### 示例数据

```
| 候选人ID        | 姓名   | 部门     | 备注       |
|-----------------|--------|----------|------------|
| U100156001     | 七分甜 | 开元商铺 | 餐饮       |
| U100156002     | 胖哥俩 | 开元商铺 | 餐饮       |
| U100156003     | 华为   | 开元商铺 | 数码       |
| U100156004     | 小菜园 | 开元商铺 | 餐饮       |
```

### 数据规则

1. **姓名列不能为空** - 空白的行会被跳过
2. **候选人ID唯一** - 如果不填，会自动生成 `CANDIDATE_行号`
3. **部门默认值** - 如果不填，使用"未知部门"
4. **数据类型** - 支持文本和数字（数字会转为字符串）
5. **行号从2开始** - 第一行是标题，从第二行开始读取

## 加载优先级

系统按以下优先级加载候选人数据：

1. **优先：Excel文件** (`lottery/prizeDate.xlsx`)
   - 如果 Excel 文件存在且包含有效数据，优先使用

2. **降级：JSON文件** (`lottery/candidates.json`)
   - 如果 Excel 加载失败或为空，尝试从 JSON 加载

3. **最后：默认数据**
   - 如果两者都失败，生成50个默认候选人

## 常见问题

### Q: Excel 文件找不到？
**A:** 确保 `prizeDate.xlsx` 文件放在 `android/app/src/main/assets/lottery/` 目录下。

### Q: 为什么没有加载我添加的候选人？
**A:** 检查以下几点：
1. 确保文件名正确（`prizeDate.xlsx`，不是 `.xls`）
2. 确保文件在 `assets/lottery/` 目录下
3. 姓名列（B列）不能为空
4. 查看日志了解加载状态

### Q: 如何查看加载了哪些候选人？
**A:**
- 在 Logcat 中过滤 "LotteryConfigManager"
- 查找日志："成功从Excel加载 X 个候选人"

### Q: 可以同时使用Excel和JSON吗？
**A:** 可以同时存在两个文件，系统会优先使用 Excel 的数据。如果 Excel 加载失败或为空，会自动降级到 JSON。

## 更新候选人数据

### 方法1：替换Excel文件
1. 编辑 `lottery/prizeDate.xlsx`
2. 重新构建应用
3. 候选人数据会自动更新

### 方法2：更新JSON文件
1. 编辑 `lottery/candidates.json`
2. 重新构建应用
3. 如果 Excel 为空，会使用 JSON 数据

## 注意事项

- ⚠️ Excel 文件必须是 `.xlsx` 格式（不支持旧版 `.xls`）
- ⚠️ 文件编码：使用 UTF-8 编码
- ⚠️ 首次加载需要网络下载 Apache POI 依赖
- ⚠️ 修改 Excel 文件后需要重新构建 APK
